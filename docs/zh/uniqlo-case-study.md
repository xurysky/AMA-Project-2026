# 优衣库（迅销集团）案例研究 — AMA Project 46

## 一、公司概况

| 指标 | 数据 |
|------|------|
| 母公司 | 迅销集团（Fast Retailing），东证/港交所上市 |
| 品牌 | UNIQLO、GU、Theory、Helmut Lang 等 |

| 年产量 | 13 亿件服装 |

## 二、为什么选优衣库做 AMA 原型

### 1. 规模够大，场景够复杂
- 全球 3,500+ 门店，13 亿件年产量
- 跨 26 国运营，亚洲市场是核心（中日韩东南亚）
- 线上+线下+第三方平台（京东、天猫）全渠道

### 2. 数字化转型走在前列
- **RFID 全面覆盖**：每件服装嵌入 RFID 芯片，实现库存实时追踪
- **"管理驾驶舱"平台**（2023年推出）：收集 3,000 万+ 条信息，实时掌握全球销售和库存
- **大数据需求预测**：自主开发算法，与工厂生产计划联动
- **门店面积扩大 10%**：关闭 30+ 低效门店，提升单店效率

### 3. AI 应用已有基础
- AI 视觉分析货架摆放，提供优化建议
- 智能客服处理退换货和尺码咨询
- 大数据驱动的需求预测和库存优化
- 但**尚未实现 Agent 化的自主决策** → 这就是我们的机会

### 4. 亚洲特殊性完美匹配课题
- 中国市场：900+ 门店，京东合作，线上线下融合
- 日本市场：RFID + 大数据 + 自动化仓库
- 东南亚市场：新兴渠道，支付碎片化
- 隐私法规：PDPA（日本 APPI、中国 PIPL、新加坡 PDPA）

## 三、现有数字化能力 vs Agent 升级空间

| 现有能力 | Agent 升级方向 |
|----------|-------------|
| RFID 库存追踪 | → **Inventory Agent**：自主预测 + 自动补货 |
| 大数据需求预测 | → **Pricing Agent**：动态定价 + 竞争响应 |
| 智能客服 | → **Customer Understanding Agent**：统一画像 + 行为预测 |
| 门店陈列优化 | → **Marketing Agent**：自主营销活动 + 预算优化 |
| 个性化推荐（基础） | → **Personalization Agent**：跨渠道千人千面 |

## 五、Agent 架构与优衣库业务映射

### Agent 1: Customer Understanding Agent
**优衣库场景：**
- 统一 2 亿+ 粉丝的线上（App/小程序/京东/天猫）和线下（900+ 门店 RFID）行为
- 合成 Persona 模拟不同客户群体（TT205）
- Agent Memory 存储跨渠道交互历史（WRK761）

### Agent 2: Personalization Agent
**优衣库场景：**
- "随心搭 省心购"栏目升级为 AI 驱动
- 基于客户画像的个性化产品推荐
- 跨渠道一致性：App 推荐 → 门店体验 → 小程序复购

### Agent 3: Inventory Agent
**优衣库场景：**
- 接管"管理驾驶舱"的 3,000 万+ 数据点
- 自主预测各门店需求，自动触发补货
- 与 Ariake Project 仓库自动化系统对接

### Agent 4: Pricing Agent
**优衣库场景：**
- 基于需求/竞争/库存的动态定价
- 中国市场：京东/天猫价格竞争响应
- 东南亚市场：支付碎片化下的价格策略

### Agent 5: Marketing Agent
**优衣库场景：**
- 自主运行微信/小程序营销活动
- 会员 2 亿+ 粉丝的精准触达
- 门店急送（顺丰同城）的营销联动

## 六、亚洲特殊性落地方案

### 隐私合规
- 日本 APPI：数据本地化 + 用户同意
- 中国 PIPL：数据出境评估 + 安全审计
- 新加坡 PDPA：跨境传输限制
- **方案**：合成 Persona（TT205）解决冷启动 + 数据本地化存储

### 多语言
- 中日韩 + 东南亚多语种
- **方案**：LLM 天然多语言能力 + Agent 语言偏好配置

### 支付生态
- 中国：微信/支付宝/京东支付
- 日本：现金/信用卡/电子支付
- 东南亚：GrabPay/GoPay/OVO 等
- **方案**：MCP 协议统一 API 接口

### 城乡差异
- 一线城市：全渠道成熟
- 三四线城市：门店扩张中（优衣库月均 7 家新店）
- **方案**：Agent 根据门店位置调整策略
